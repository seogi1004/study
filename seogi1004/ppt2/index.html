JavaScript Functions

Alvin Hong
July 4, 2015

Function Declaration (FD)

Features

  • 함수 이름을 설정해야 함
  • 변수 객체(VO)에 영향을 미침
  • 컨텍스트 진입시 생성됨

function exampleFunc() {
  ...
}

컨텍스트 내의 모든 위치에서 실행 가능 (Hoisting)

foo();

function foo() {
    alert('foo');
}

FD 위치에 따라 컨텍스트가 달라짐

// function can be declared:
// 1) directly in the global context
function globalFD() {
  // 2) or inside the body
  // of another function
  function innerFD() {}
}

QA

FD은 컨텍스트 진입시 생성되는 특성이 있음

if (true) {
  function foo() {
    alert(0);
  }
} else {
  function foo() {
    alert(1);
  }
}

foo(); // 1 or 0 ? test in different implementations

Function Expression (FE)

Features

  • 함수 이름을 설정하지 않아도 됨
  • 변수 객체(VO)에 영향을 미치지 않음
  • 코드 실행 단계에 생성됨

익명 함수(anonymous FE)를 foo 변수에 할당함

var foo = function() {
  ...
};
var foo = function _foo() {
  ...
};

FE는 다음과 같이 사용할 수 있음

// in parentheses (grouping operator) can be only an expression
(function foo() {});

// in the array initialiser – also only expressions
[function bar() {}];

// comma also operates with expressions
1, function baz() {};

FE는 코드 실행 단계에서 생성되고, VO에 저장되지 않음

alert(foo);

(function foo() {});

alert(foo);

Encapsulation

캡슐화 된 스코프의 생성은 외부 컨텍스트로부터 데이터를 숨길 수 있다.

var foo = {};

(function initialize() {
  var x = 10;

  foo.bar = function () {
    alert(x);
  };
})();

foo.bar();
alert(x);

Why use parentheses ()

  • 표준에 따르면 표현식은 중괄호로 시작할 수 없음
  • 함수 선언과 구분하기 위함

아래 코드는 모두 에러가 발생함

function () {
  ...
}();

// or even with a name
function foo() {
  ...
}();

Kind of parenthesis ()

괄호는 표현식과 그룹 연산자 두가지 형태가 있기 때문에 이를 구분할 수 있어야 함

// "foo" is a function declaration
// and is created on entering the context

alert(foo); // function

function foo(x) {
  alert(x);
}(1); // and this is just a grouping operator

foo(10);

괄호는 표현식과 그룹 연산자 두가지 형태로 사용

// function declaration
function foo(x) {
  alert(x);
}

// a grouping operator
// with the expression
(1);

// another grouping operator with
// another (function) expression
(function () {});

// also - the expression inside
("foo");

Advanced

자바스크립트 파서가 함수 표현 위치를 알고 있다면 괄호는 필요 없음

var foo = {
  bar: function (x) {
    return x % 2 != 0 ? 'yes' : 'no';
  }(1)
};

alert(foo.bar);

다음과 같은 방법으로 함수를 FE로 변환시킬 수 있음

1, function () {
  alert('anonymous function is called');
}();

// or this one
!function () {
  alert('ECMAScript');
}();

하지만 괄호를 사용하는 것이 가장 좋은 방법이라고 알려져 있음

(function () {})();
(function () {}());
